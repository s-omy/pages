# ファクトチェックレポート：Codex + cc-sdd 仕様駆動開発ベストプラクティス

**確認日:** 2026-07-08
**対象ドキュメント:** `cc_sdd.md`

---

## 1. 全体評価

本レポートは、業務共通機能を持つビジネスフレームワーク上の開発において、AIコーディングエージェントを安全かつ効率的に運用するための極めて実用的なガイドラインとして機能しています。

レポート内で言及されている前提技術（Codex CLI、AGENTS.md、cc-sdd、MCP等）は、すべて2026年現在の事実・仕様と整合しています。特に、長大なコンテキストを与えることによるAIの精度低下（Context Rot）や、過剰な自律行動（Overeager Actions）といった既知の問題に対し、仕様駆動開発（SDD）のアプローチで境界を定義する解決策は技術的に妥当です。

## 2. 主要技術要素のファクトチェック

レポートを構成する中核的な技術・ツールについて、現在の公開情報と照合した結果は以下の通りです。

### 2.1 Codex CLI / Codex app
* **事実確認:** 妥当
* **根拠:** Codex CLIは、2025年に正式公開されたOpenAI公式のターミナル型コーディングエージェントです。ターミナル上で動作し、コードの読解や編集、コマンド実行をローカル環境で直接行う仕様となっています。
* **評価:** ファイル内での「ローカル端末で動くcoding agent」「選択ディレクトリ内のコードを読み、変更し、コマンドを実行できる」という説明は、Codex CLIの実際の挙動と一致しています。

### 2.2 AGENTS.md
* **事実確認:** 妥当
* **根拠:** `AGENTS.md` は、2025年8月にOpenAIを中心とした複数のAI開発企業が協力して公開した、AIコーディングエージェント向けの標準化された指示フォーマット（AI向けREADME）です。
* **評価:** ファイル内で言及されている「階層化できる」「参照先インデックスや検証コマンドを置く」という利用方法は、人間向けの `README.md` とエージェント向けの `AGENTS.md` を分離し、指示を簡潔に保つという公式の設計思想に沿っています。

### 2.3 cc-sdd (gotalab)
* **事実確認:** 妥当
* **根拠:** `cc-sdd` は gotalab によって開発されている Kiro 互換の仕様駆動開発フレームワーク（OSS）です。v3系では、`/kiro-discovery` を新しいエントリーポイントとし、Agent Skills を統合した自律実装の制御に重点を置いています。
* **評価:** ファイルに記載のある `/kiro-discovery`、`/kiro-impl`、および要件からタスクに分解してAIを制御するワークフローの説明は、`cc-sdd` の最新仕様と完全に一致します。

### 2.4 Model Context Protocol (MCP)
* **事実確認:** 妥当
* **根拠:** MCPは、AIアプリケーションと外部のツールやデータベースを接続するためにAnthropicなどが提唱しているオープン標準プロトコルです。
* **評価:** ファイル内での「外部ドキュメントやAPI仕様などの取得に有効」「無制限の本番操作には使わない」という整理は、MCPのセキュリティ（Trust & Safety）の観点から非常に正確な判断です。

## 3. ベストプラクティスと戦略の妥当性評価

レポート内で提示されている「効くTIPS」や運用戦略について、AIエージェントの挙動特性の観点から評価しました。

### 3.1 「ルールID化」と「Applicable Rules」の事前提示
* **評価:** 非常に有効
* AIはフレームワークの独自仕様を知らないため、一般的な実装（例：素のSpring Boot）に倒れがちです。実装前にルールIDと適用可否の判断を強制することで、AIの注意機構（Attention）を特定の規約にアンカーさせ、フレームワークを逸脱するリスクを大幅に下げることができます。

### 3.2 `_Boundary:_` による責務境界の宣言
* **評価:** 必須レベルの安全策
* AIコーディングエージェントが依頼の範囲を超えて関係のないファイルを修正してしまう「Overeager Actions」は、現在の自律型エージェント共通の課題です。`tasks.md` 内で対象境界と「触ってはいけないファイル（no-touch list）」を明記することは、この問題に対する最も直接的で効果的な防御策です。

### 3.3 トークンコストと Context Rot の回避
* **評価:** 妥当
* 一度に長文の規約をプロンプトに含めると、AIが重要な制約を見落とす確率が上がります。レポートが提唱する「AGENTS.mdを短くし、詳細は参照化する」「1スレッド1タスク」という原則は、コンテキストウィンドウの消費を抑えつつ回答精度を保つための最適解です。
